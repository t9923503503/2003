# 👑 IPT Mixed — Руководство по формату

## Что такое IPT Mixed?

**IPT Mixed (Individual Point Tracker)** — турнирный формат для пляжного волейбола, где игроки соревнуются в парах (2v2), но очки считаются **индивидуально**. Напарники меняются каждый раунд по матрице ротации. Итоговый рейтинг: Победы → Разница → Очков.

---

## Как запустить IPT — Быстрый старт из РОСТЕРА

Не нужно создавать турнир отдельно. Всё делается прямо из раздела **⚙️ РОСТЕР**:

1. Открыть **РОСТЕР** → карточка «Формат турнира»
2. Нажать таб **👑 IPT Mixed**
3. Выбрать **количество кортов** (1 / 2 / 3 / 4)
4. Выбрать **лимит очков** (10 / 12 / 15 / 18 / 21)
5. Выбрать **тип финиша**: Хард или ±2 Баланс
6. Отметить **участников** из базы игроков (с поиском по имени)
7. Нажать **🏐 Запустить IPT**

> Система автоматически создаёт турнир `ipt_quick`, делит игроков на группы и открывает судейский интерфейс.

## Матрица ротации (8 игроков, 4 раунда, 2 корта)

| Раунд | Корт А          | Корт Б          |
|-------|-----------------|-----------------|
| 1     | [0,1] vs [2,3]  | [4,5] vs [6,7]  |
| 2     | [0,2] vs [4,6]  | [1,3] vs [5,7]  |
| 3     | [0,4] vs [1,6]  | [2,7] vs [3,5]  |
| 4     | [0,7] vs [2,5]  | [1,4] vs [3,6]  |

**Свойства:** каждый игрок играет с 4 разными напарниками. Встреч с одним соперником — не более 2 раз.

## Ведение счёта (мобильный интерфейс)

Каждый К-таб показывает карточки кортов своей группы:

```
┌──────────────────────────────────┐
│ 🏅 КОРТ A                       │
├──────────────────────────────────┤
│ Иванов + Петров   [−]  21  [+]  │  ← победитель (зелёный фон)
├──────────── VS ──────────────────┤
│ Сидоров + Кузнецов [−]  14  [+] │
└──────────────────────────────────┘
```

- Кнопки **[+]** и **[−]** — 60×60px, удобны на телефоне
- При достижении лимита матч завершается автоматически: **✅ ЗАВЕРШЕНО**
- После завершения всех кортов раунда: кнопка **▶ Следующий раунд**
- После последнего раунда: кнопка **🏁 Завершить [группа]**

---

## Кросс-таблица (результаты группы)

Под кортами каждой группы отображается живая таблица:

| # | Игрок | Р1 | Р2 | Р3 | Р4 | В | WR | ± | Оч |
|---|-------|----|----|----|----|---|----|---|----|
| 🥇 | Иванов | 21 | 18 | — | 15 | 3 | 100% | +18 | 54 |
| 🥈 | Петров | — | 21 | 15 | 21 | 2 | 67% | +12 | 57 |
| 🥉 | Морозов | 21 | 14 | 18 | — | 2 | 67% | +8 | 53 |

**Обозначения колонок:**
- **Р1–Р4** — очки игрока в каждом раунде (прочерк = скамейка)
- **В** — победы
- **WR** — процент побед
- **±** — разница очков (свои − чужие)
- **Оч** — сумма своих очков

**Сортировка:** В → ± → Оч

---

## Завершение

### Завершение группы
После 4-го раунда → кнопка **🏁 Завершить [ХАРД]** → группа помечается `finished`.

### Завершение турнира
Когда **все группы** завершены → кнопка **🏆 Завершить турнир** → результаты записываются в рейтинг игроков.

## Как это работает внутри (для разработчиков)

### Главный принцип: переиспользование существующих кнопок

Отдельного экрана для IPT **нет**. Вместо этого в функции `_switchTabInner` добавлен перехват:

```js
// Нажатие К1 (id=0) при активном IPT:
if (_iptTrn?.ipt?.groups) {
    if (typeof id === 'number' && _iptTrn.ipt.groups[id]) {
        screen.innerHTML = renderIPTGroup(id); // ← вместо renderCourt()
        return;
    }
}
```

### buildNav() — динамическое количество кнопок

```js
// Количество К-пиллов = количество IPT групп (не nc)
const courtCount = iptGroups ? iptGroups.length : nc;

// Финальные кнопки тоже динамические
const divKeys = iptGroups
    ? getIPTFinalsNavKeys(iptGroups.length)  // IPT: HD+LT, HD+MD+LT...
    : activeDivKeys();                        // стандарт
```

### Функция getIPTFinalsNavKeys(n)
```js
// 1 группа  → ['hard']
// 2 группы  → ['hard', 'lite']
// 3 группы  → ['hard', 'medium', 'lite']
// 4 группы  → ['hard', 'advance', 'medium', 'lite']
```
